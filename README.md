# MatchDay — AI Game Day Companion

> From ticket to final whistle. Your personal AI concierge for large-scale sporting events.

Built for PromptWars Virtual by Hack2Skill x Google for Developers.

---

## What it does

1. **Scan your ticket** — Upload any match ticket. Gemini Vision extracts all details in seconds.
2. **Travel planning** — Tell us where you're coming from. Gemini reasons over kickoff time, crowd patterns, and gate timing to give you a leave-by time with full rationale.
3. **Venue intel** — Food stalls, gates (with live congestion), restrooms, and medical points — all personalised to your seat.
4. **Game day plan** — A structured timeline from "leave home" to "post-match exit", generated by Gemini based on everything it knows about you.
5. **Live AI concierge** — Multi-turn chat with full context. Ask anything mid-match and Gemini updates your plan.

---

## Stack

- **Frontend**: Next.js 14 (App Router), Tailwind CSS, Framer Motion
- **AI**: Gemini 1.5 Pro (ticket OCR), Gemini 1.5 Flash (travel + plan + chat)
- **Maps**: `@vis.gl/react-google-maps` — Google Maps with route rendering, facility markers, filter chips
- **State**: Zustand (full game day state across 5 stages)
- **Deploy**: Vercel (Next.js only, no backend)

---

## API keys needed

### Gemini API key
1. Go to [Google AI Studio](https://aistudio.google.com)
2. Create API key → copy to `GEMINI_API_KEY`

### Google Maps API key
1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Enable these 4 APIs:
   - Maps JavaScript API
   - Places API
   - Routes API (for route rendering)
   - Geocoding API
3. Create an API key → copy to `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`
4. Restrict the key to your domain in production

## Project structure

```
app/
  layout.tsx              # Google Maps APIProvider, fonts, dark theme
  page.tsx                # Stage orchestrator (5 stages, animated transitions)
  globals.css             # Design tokens, glass utility, glow effects
  api/
    extract-ticket/       # POST — Gemini Pro multimodal OCR
    travel-plan/          # POST — Gemini Flash travel reasoning
    plan-generate/        # POST — Gemini Flash game day plan
    venue-chat/           # POST — Multi-turn stateful chat

components/
  ticket/TicketUpload     # Drag/drop + scan animation
  onboarding/             # 5-step preferences wizard with Places autocomplete
  travel/TravelScreen     # Gemini travel options + live route map
  venue/VenueScreen       # Facility cards + venue map with markers
  plan/PlanScreen         # Timeline + floating AI chat drawer
  maps/VenueMap           # Reusable map component with route + markers
  chat/StadiumChat        # Multi-turn chat with quick prompts

lib/
  store.ts                # Zustand store (all game day state)
  gemini.ts               # Flash + Pro clients, JSON parser, image builder

data/
  venues.ts               # Static venue data (Wankhede, Eden Gardens)
```

## Prompt library (key prompts)

| Prompt | File | Purpose |
|---|---|---|
| `TICKET_EXTRACTION_PROMPT` | `api/extract-ticket/route.ts` | Multimodal OCR → structured JSON |
| `TRAVEL_PLAN_PROMPT` | `api/travel-plan/route.ts` | Timing-aware travel reasoning |
| `SYSTEM_CONTEXT` | `api/venue-chat/route.ts` | Multi-turn concierge context |
| Plan generation prompt | `components/plan/PlanScreen.tsx` | Full timeline generation |

## Adding a new venue

Add an entry to `data/venues.ts` following the existing `VenueInfo` schema. The venue name in the ticket extracted by Gemini is fuzzy-matched against keys in this object.
